### A2. Broken theme system
- `apps/web/app/layout.tsx:27` ‚Äî `<html lang="en" data-theme="light">` hardcoded; no toggle exists anywhere.
- `apps/web/app/globals.css:86` ‚Äî `[data-theme="dark"], :root { ‚Ä¶ }` makes **dark** the `:root` default, conflicting with the hardcoded light attribute.
- Reader preferences (`QuranReaderClient.tsx:16`) default `theme: 'dark'` but the value is never applied to `<html>`.
- Settings page theme row is display-only.
- **Fix:** Phase 1 ThemeProvider (see `02-DESIGN-SYSTEM.md` ¬ß5).

### A3. Dead fetch to non-existent API (home page)
- `apps/web/app/page.tsx:40` ‚Äî `fetch('/api/verse?surah=‚Ä¶&ayah=‚Ä¶')`. The app is a **static export** (`next.config.ts` ‚Üí `output: 'export'`); there are no API routes and none exist in `app/`. On web and Android this always fails ‚Üí "Continue Reading" card permanently shows the fallback (Surah 18, juz shows "Loading‚Ä¶" replaced by fallback).
- **Fix:** Phase 2 ‚Äî read verse data from a bundled static JSON (per-surah JSON emitted at build) or embed last-read text preview in localStorage when the user leaves the reader.
- **Same class of bug in `/ai`:** `apps/web/app/ai/page.tsx:36-42` fetches `/api/ai/ask-quran|ask-hadith|topic` ‚Äî none exist in a static export; the page silently does nothing on submit. Phase 2 converts it to a clearly-labeled "Coming soon" screen (backend AI needs `apps/api` deployment ‚Äî out of current scope). Note: `/search` and `/quran/compare` fetch `/search-index.json`, which IS a bundled static asset generated by `scripts/build-search.js` ‚Äî that pattern is correct, keep it.

### A4. Navigation defects (layout.tsx)
- `layout.tsx:94` ‚Äî `const isActive = link.id === 'home';` ‚Üí bottom-nav active state is **hardcoded to Home** forever.
- `layout.tsx:15,22` ‚Äî nav links to **`/duas` which has no page** ‚Üí 404. Also in the footer (`:82`).
- `layout.tsx:61-63` ‚Äî EN/BN language toggle is two static `<button>`s in a **server component**; no handler, does nothing.
- The layout is a server component so none of these can be interactive without extracting client components.
- **Fix:** Phase 2 ¬ß3.

### A5. Google Fonts loaded from the network
- `apps/web/app/globals.css:1` ‚Äî `@import url('https://fonts.googleapis.com/...')` for Inter, Hind Siliguri, Noto Naskh Arabic, Noto Sans/Serif Bengali.
- **Impact:** on Android offline, ALL custom fonts fail ‚Üí Arabic renders in a fallback system font (layout shifts, ugly Uthmani rendering); Bengali may render tofu on some devices. Also a render-blocking request on web.
- **Fix:** Phase 1 ¬ß4 ‚Äî self-host via `next/font` (bundles fonts into the static export).

## B. High ‚Äî Android/Capacitor gaps

### B1. No Capacitor plugins installed at all
- Root `package.json`: only `@capacitor/core 6.2.0`, `@capacitor/android 6.2.1`, `@capacitor/cli 6.2.1`.
- `android/capacitor.settings.gradle` includes only `capacitor-android`. No Filesystem (needed for offline audio), no StatusBar, no SplashScreen, no Network, no App (back-button handling).
- **Version mismatch:** core `6.2.0` vs android/cli `6.2.1` ‚Äî align all to the same 6.x patch.

### B2. No safe-area handling
- Bottom nav (`layout.tsx:90`) is `fixed bottom-0` with `pb-3` ‚Äî on gesture-nav Android devices it collides with the system gesture bar. Header has no `env(safe-area-inset-top)` consideration. No `viewport-fit=cover`.
- **Fix:** Phase 1 ¬ß6 + Phase 3.

### B3. AndroidManifest is bare
- `android/app/src/main/AndroidManifest.xml` ‚Äî only `INTERNET` permission. That is actually fine for the planned feature set (app-private storage needs no permission), but the manifest lacks `android:usesCleartextTraffic` consideration and there is no config for keeping audio alive; documented in Phase 3.

### B4. No build pipeline for Android
- No script chains `web build ‚Üí cap sync`. Root scripts only have turbo `dev`/`build`. `capacitor.config.json` correctly points `webDir: apps/web/out` and a full export exists, but it can silently go stale.
- **Fix:** Phase 3 ¬ß5 adds `build:android` script.

## D. What already works (do not break)

- Static export pipeline: `generateStaticParams` for all 114 surah pages, 6236 tafsir pages, hifz pages; `apps/web/out/` fully generated (~70k files).
- Audio streaming: per-ayah MP3s from `https://everyayah.com/data/{reciterId}/{SSS}{AAA}.mp3` (`components/quran/audio-utils.ts:13-15`), reciters: Alafasy_128kbps (default), Ghamadi_40kbps, Husary_128kbps. Auto-advance playlist, seek, volume, rate ‚Äî solid foundation for the offline layer.
- IndexedDB layer (`lib/storage/indexeddb.ts`): DB `AlQuranOfflineDB` v1, stores `surahs`, `verses`, `bookmarks`, `settings`. Bookmarks/last-read/recent-searches helpers.
- Reading preferences via `useLocalStorage('alquran_preferences')` with font-size multipliers wired to CSS vars.
- Hifz mode, tafsir pages, compare view, library pages (bookmarks/notes/highlights/collections), hadith list pages ‚Äî functional logic, broken styling only.
- Home page (`app/page.tsx`) and root layout are already Tailwind-styled and look premium (emerald `#0a3622` + gold `#c5a059` "Majestic" theme) ‚Äî this is the visual reference for the rest of the app.

## Implementation Plan: Al_Quran Flutter App

### Project Placement
Create **`apps/flutter/`** inside the monorepo as a standalone Flutter project. It coexists with the existing pnpm/Turbo apps.

### Data Strategy (Offline-First)
Bundle all `data/*.json` files as Flutter **assets** for fully offline-first operation. Port the TypeScript merge logic from `apps/web/lib/data/quran.ts` into Dart. This satisfies offline requirements from day one.

### Porting From Existing Codebase
- `packages/types/` → Dart `equatable` data classes (Surah, Ayah, Bookmark, etc.)
- `packages/tokens/` → Flutter ThemeData + ColorScheme
- `packages/quran/` → Pure Dart utility functions
- `packages/search/` → Dart string normalization + highlight
- `data/*.json` → Flutter AssetBundle + Hive caching

### Phases

**Phase 1 — Foundation**
1. `flutter create al_quran --platforms=android,ios,web` in `apps/flutter/`
2. Configure pubspec.yaml with all dependencies (flutter_bloc, go_router, dio, hive_flutter, just_audio, audio_service, google_fonts, equatable, shimmer, etc.)
3. Set up folder structure: lib/config/, lib/core/, lib/features/, lib/shared/
4. Port design tokens → light/dark ThemeData with Islamic palette
5. Set up go_router with all named routes
6. Create Dio API client with interceptors (base URL: alquran.cloud)
7. Initialize Hive: settingsBox, bookmarksBox, lastReadBox, cacheBox

**Phase 2 — Surah List & Home**
1. Parse data/surahs.json as asset → Surah models
2. Home screen: greeting, "Continue Reading" card, Daily Ayah, quick-access grid
3. Surah List: ornamental number tiles, Arabic/English names, revelation badges, search/filter chips
4. Juz List: 30 cards with start/end references

**Phase 3 — Reading View (Core)**
1. Merge 5 JSON sources (Arabic + Bangla + English + Transliteration + Pronunciation) by array index
2. Bismillah header (ornate, centered) for all except At-Tawbah
3. Ayah cards: Arabic Uthmani (Amiri font), verse number ornament, translation layers, transliteration
4. Reading toolbar: font sliders, layer toggles, reading mode, theme toggle
5. Swipe between surahs, "Jump to Ayah" picker

**Phase 4 — Bookmarks & Last Read**
1. Hive-based bookmarks: {surahNumber, ayahNumber, surahName, timestamp, note}
2. Bookmarks screen with search, swipe-to-delete, navigation to ayah
3. Auto-save last read position on exit

**Phase 5 — Audio Player**
1. just_audio + audio_service for background playback
2. Stream from everyayah.com (Alafasy, Ghamadi, Husary reciters)
3. Persistent bottom audio bar with progress, play/pause, prev/next, reciter selection
4. Ayah highlighting synced with audio position

**Phase 6 — Search** (client-side over bundled data)
**Phase 7 — Tafsir** (Quran.com API v4)
**Phase 8 — Settings & Onboarding** (full settings + 5-screen onboarding flow)
**Phase 9 — Caching & Offline** (Hive cache with 7-day TTL, download UI)
**Phase 10 — Platform Features** (Android widget, share ayah image, PWA, keyboard shortcuts)
**Phase 11 — Polish** (animations, shimmer, smooth scroll, lazy loading)
**Phase 12 — Testing & Deployment** (unit/widget/integration tests, Play Store, Firebase Hosting)

### Starting Point
Begin with Phase 1: create the Flutter project, set up folder structure, configure themes, routing, API client, and Hive initialization. Then proceed sequentially through the phases.

### Decisions
- Use Amiri font for Arabic (as plan specifies)
- Include iOS (Flutter default, zero cost)
- Latest stable dependency versions (not outdated pins)
